# 🗄️ Archive

冷藏起来的东西 —— 不再活跃但留存以备日后查阅。

## 干啥用的

Archive 用来存：

- 已完成的项目和它们的产出
- 不再维护的 area
- 留作历史参考的旧笔记
- 弃用的资源
- 过去的实验和迭代版本

## 什么该放这里

### 来自 Projects

- 已完成项目（连同最终交付物）
- 已取消项目（连同教训）
- 30 天以上不活跃的项目

### 来自 Areas

- 不再相关的 area
- 已交接出去的责任
- 已不再担任的角色

### 来自 Resources

- 过时但有历史价值的信息
- 被替代的框架或方法
- 旧版本的演化想法

### 来自 Inbox

- 处理过、不再需要的条目
- 旧的 daily notes（提炼后）
- 没有长期价值的随手记录

## 组织方式

```
04_Archive/
├── Projects_2024/     # 按年归档的已完成项目
├── Projects_2023/
├── Old_Areas/        # 已结束的 area
├── Daily_Notes/      # 旧的每日记录
├── Ideas/           # 没发展起来的想法
└── Miscellaneous/   # 其他
```

## 归档流程

### 归档项目前

1. 写完成 summary
2. 把可复用的洞察抽到 Resources
3. 记录学到的教训
4. 更新相关的 area
5. 整体目录结构原样移动

### 完成 summary 模板

```markdown
# 项目: [Name] - 完成总结

**周期**：起始日 - 结束日 **状态**：完成 / 取消 / 暂停

## 目标

- 原目标 1 ✓
- 原目标 2 ✓
- 原目标 3 ✗

## 关键产出

- 交付了什么
- 产生了什么影响
- 创造了什么价值

## 教训

- 哪些做得好
- 哪些没做好
- 下次该怎么改

## 可复用的资产

- 创建的模板：[[link]]
- 沉淀的流程：[[link]]
- 获得的洞察：[[link]]

## 相关笔记

- 延续到：[[Area name]]
- 另见：[[相关项目]]
```

## Claude Code 工作流

### 归档项目

```
帮我归档 [项目名]。
做完成总结。
把可复用洞察抽到 Resources。
全部移到 Archive。
```

### 搜索归档

```
在 archive 里搜 [话题]。
我需要历史上下文。
```

### 年度回顾

```
看下 [年份] 我归档的所有项目。
你看到什么模式？
我达成了什么？
```

### 复活项目

```
我想重启 [已归档项目]。
归档时是什么状态？
需要更新什么？
```

## 归档哲学

### 它不是坟场

- 归档保留组织记忆
- 旧项目能启发新项目
- 模式会在时间里浮现
- 想法可以被复活

### 它也不是垃圾场

- 归档要走心
- 保持基本组织
- summary 要可访问
- 真垃圾该删就删

## Claude Code 提示词

### 历史分析

```
看下我归档的项目。
我倾向于开始但不完成什么类型的事？
有哪些反复出现的主题？
```

### 知识挖掘

```
在 archive 里搜下 [概念] 的所有提及。
我的想法如何演变？
```

### 模式识别

```
分析我的项目完成率。
什么因素和成功相关？
什么模式预示失败？
```

## 维护节奏

### 季度

- 看下最近新增
- 确认 summary 都在
- 检查有没有可复活的项目

### 年度

- 大整理
- 删掉真死的东西
- 抽取漏掉的洞察
- 必要时重组

## 几条建议

- **加日期** — 未来的你会感激你
- **总是写 summary** — 上下文消失得很快
- **大胆加链接** — 关联能熬过归档
- **常搜常用** — Archive 是用来用的
- **果断删** — 不是所有东西都值得留

## 记住

Archive 是你的组织记忆。它的意义不是"什么都留着"，而是"保留那些可能启发未来工作的东西"。让 Claude Code 帮你看到时间维度上的模式，从经验里提炼智慧。
