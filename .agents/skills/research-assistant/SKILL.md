# Research Assistant

通过搜 vault 并综合发现，对话题做深入调研。

## 流程

1. **初步搜索**
   - 在整个 vault 搜该话题
   - 识别所有相关笔记
   - 注意已有知识的缺口

2. **深入挖**
   - 仔细读所有相关笔记
   - 抽出关键洞察和引用
   - 找出矛盾或张力
   - 画出想法之间的连接

3. **综合**
   - 总结发现
   - 突出模式和主题
   - 标注未解的问题
   - 建议进一步研究方向

## 输出结构

```markdown
# 调研总结：[话题]

## 已有知识

- vault 里已经有什么
- 之前工作的关键洞察

## 关键主题

1. 主题 1
   - 支撑笔记：[[note1]], [[note2]]
   - 关键洞察：...
2. 主题 2
   - 支撑笔记：[[note3]], [[note4]]
   - 关键洞察：...

## 矛盾 / 张力

- 想法冲突的地方
- 未解问题

## 缺口

- 缺什么
- 接下来要研究什么

## 连接

- 相关话题：[[topic1]], [[topic2]]
- 意外的关联：...

## 建议下一步

1. 需要做的具体研究
2. 要探索的问题
3. 可以做的实验
```

## 几条建议

- 一开始撒大网，再聚焦
- 留意意外的连接
- 别忽视矛盾 — 洞察经常藏在那里
- 永远建议具体的下一步
