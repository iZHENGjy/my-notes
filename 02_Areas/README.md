# 🔄 Areas

长期持续的责任和活动领域。

## 干啥用的

Areas 是**持续性责任**，特征：
- 没有结束日期
- 需要持续投入
- 定义了你的角色和承诺
- 需要定期维护

## 什么该放这里

### 个人 Area
- 健康与体能
- 财务
- 人际关系
- 个人成长
- 家庭与环境

### 职业 Area
- 团队管理
- 客户关系
- 专业技能
- 行业知识
- 人脉网络

### 创作 Area
- 写作训练
- 内容创作
- 学习与研究
- 业余项目

## Area 结构

每个 area 大致这样：
```
Area_Name/
├── README.md          # 概览和原则
├── Goals/            # 当前关注点
├── Resources/        # 参考资料
├── Reviews/          # 定期复盘
└── Ideas/           # 未来可能性
```

## Claude Code 工作流

### Area 复盘
```
看下我的 [area 名] area。
最近的笔记里有什么模式？
哪些方面需要更多关注？
```

### 设定目标
```
帮我给 [area] 设季度目标。
基于近期活动，应该优先做什么？
```

### 找盲区
```
分析我的 [area] 笔记。
有哪些重要话题我没在追？
有哪些问题我应该问自己？
```

### 跨 Area 分析
```
对比我的 [Area A] 和 [Area B] 笔记。
重叠在哪？
怎么互相借鉴？
```

## Area vs Project

**问自己**：这事有没有结束日期？

| Area（长期） | Project（限时） |
|---------------|---------------------|
| "健康" | "6 月前减 10 斤" |
| "财务" | "做 2024 年预算" |
| "写作" | "写完一个博客系列" |
| "学习" | "完成 Python 课程" |
| "客户" | "交付项目 X" |

## 维护节奏

### 每周
- 快速扫一遍每个 area
- 标记紧急事项
- 记下新想法

### 每月
- 复盘目标和进度
- 更新参考资料
- 清理旧笔记

### 每季度
- 用 Claude Code 深度复盘
- 调整重心
- 归档过时材料

## Claude Code 提示词

### 健康检查
```
对我所有 area 做次健康检查。
哪些最近有活动？
哪些被冷落了？
```

### 平衡评估
```
看我的 area，最近精力主要花在哪？
哪里失衡了？
```

### 整合机会
```
找下我不同 area 之间的联系。
哪些地方更好整合能创造价值？
```

## 几条建议

- **Area 引出 Project** — 项目常常从 area 的需求里冒出来
- **保持活跃** — 死掉的 area 该归档
- **定期回看** — area 不打理就会漂移
- **写下标准** — "做得好"是什么样子？
- **量化跟踪** — 哪怕定性的 area 也能用指标辅助

## 记住

Area 是你生活的背景板。它们不会"完成"，但会演化。让 Claude Code 帮你看到模式，在所有责任之间保持平衡。
